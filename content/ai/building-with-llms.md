# Building with LLMs

My default approach is:

1. Start with the user workflow (what “done” looks like).
2. Make the system observable early (logs, evaluation sets, and simple metrics).
3. Keep the first version boring (fewer moving parts, fewer prompts).
4. Add structure only when it pays for itself (tools, retrieval, routing, memory).

Topics I plan to cover:

- Prompting patterns that hold up under iteration
- Retrieval (RAG) that doesn’t become a second product
- Evaluation: test cases, regressions, and “what good looks like”
- Tool use and agent design for real tasks
